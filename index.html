Recent works

Robotics

Automated Security

I set up ROS 2 Galactic on windows, running a websocket with a live video feed from a robot capable of doing navigation around a perimeter using rigidly defined rules for navigation however lacking path planning, written in Python.

This project was run in a scrum team, adhering to strict routines independent of external oversight, with a large documentation and report written at the end and an accompanying video tutorial on running the software.

The footage of the robot was entirely from the simulator VREP as no real-world robot could be acquired to prototype with.

Automated Exploration

This project was written inside of a deployed Docker running Ubuntu, with the program being written in Python. The main navigation algorithm was the RRT algorithm, with the navigation stack being run in the background to produce a robot capable of advanced planning while performing SLAM.

The slam-toolbox was included in the plugins for the project to help create a map and to draw map meta-data into the python script and OpenCV was used for the image processing requirements for segmenting relevant landmarks and identifying them.

An accompanying markdown file to direct the user is included alongisde a narrated video explaining the topic including things such as the computational complexity.

Gazebo was used as a simulator as the main robot needed was the turtlebot 3 and it was not available for this project.

Artificial Intelligence and Machine Learning

Big Data Solution

Utilising the Hadoop ecosystem, and Google Collab to deliver several ML solutions to common problems, ranging from identification of potential issues in Nuclear Reactor sensors, to identifying and analysing flight and passenger information on a large scale, I am comfortable using these tools to create a clean solution to a wide variety of issues. My main tool utilised was PySpark as that is the tool that I am most comfortable with, using mapping and reducing functions to deliver organised and relevant information, with ML models to predict trends upon the data.

Meta-Learning as a secondary diagnostic aid

For my dissertation I produced a Meta-Learning algorithm, capable of diagnosing cancer after training on a dataset of 20,000 images. This model was able to diagnose lung and colon cancers from histeopathographs, with training times under 2 hours.

Included in this project was an indepth exploration of the topic with a large and varied range of research into different related subtopics. Another included section was the scalable system for comparing training over epochs and statistics calculations.

Despite the complexity of overfit mitigation and evaluation, there was a minor amount of overfitting mitigation and a larger emphasis on evaluation through methods such as confusion matrices.

Fraud detection and data cleansing

Another project is my automated debit card fraud detection system, firstly imputating and cleaning data before using diagnostic analysis and PCA to analyse and extract relevant features from the data lake, and then further develop and evaluate the model. There is also a dictionary and counter system tracks independent customers and their previously interacted accounts.

Drawing from a large amount of data, this would ideally be solved through the use of the Hadoop ecosystem, such as PySpark. The main focus of this project was to build a python pipeline involving generators and decorators to help with different aspects of forming a data pipeline.

Missing Data imputation experiments

I experimented with using GANs and few-shot learning to create a model that can generate custom data imputation on varying sizes of databases with missing data, allowing for a cheap and fast alternative to common methods such as mean imputation or mode imputation.

While proving difficult, GANs show some potential when combined with the few-shot paradigm to create a resulting set of models that attempt to tackle the issue, with varying levels of success.
